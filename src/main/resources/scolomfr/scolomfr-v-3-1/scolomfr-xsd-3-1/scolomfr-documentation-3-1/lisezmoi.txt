Ce fichier est un élément de l'ensemble des schémas XSD nécessaires à la mise en œuvre de ScoLOMFR 3.1 (décembre 2016).
ScoLOMFR est un profil d'application de LOM et LOMFR.

La version 3.1 des schémas intègre la validation de l'ensemble des vocabulaires définis à cette date, en reprenant les ajouts suivants :
        - intégration d'un élément complexe 4.3.1 "localisation étendue", destiné à fournir les éléments nécessaires pour un accès à la ressource via des plates-formes de gestion des accès ;
        - intégration des URI comme identifiants de tous les concepts utilisés, dans une évolution vers le web sémantique ;
        - ajout d'un élément "label" à tous les éléments déclinés du lom, et définis comme des paires (source, value).
                   - Le champ "source" devient SCOLOMv3.1 pour cette version ;
                   - le champ "value" porte l'URI du concept, tel que défini dans les vocabulaires publiés ;
                   - le champ "label" permet de saisir un libellé (par défaut le libellé du terme préférentiel du concept).
          Les présents schémas XSD présentent une compatibilité ascendante avec les fichiers conformes à LOMv1.0, LOMFR et ScoLOMFRv1.x., ScoLOMFRv2.x. et ScoLOMFRv3.0.
          Toutes les valeurs de termes ayant été valides dans les schémas et vocabulaires publiés pour ces différentes versions sont valides au sens des schémas ScoLOMFRv3.1.
         Toutefois, depuis la publication de ScoLOMFRv2.0, le présent package propose aussi un outil de migration permettant de migrer les fichiers LOM, LOMFR et ScoLOMFRv1.x en un format "strict" ScoLOMFRv3.1, dans lequel tous les éléments liés à des vocabulaires sont identifiés par leurs URI de concepts.
         L'ensemble des vocabulaires, de la documentation et des éléments de cycle de vie de ScoLOMFR sont publiés sur le site https://www.reseau-canope.fr/scolomfr/.

XSD ScoLOMFR 3.1 intégrant :

- l'ajout de l'élément 4.3.1 extendedLocation
- la modification des balises <lom:language> pour supporter une URL de langue
- la modification de la balise <lom:format> pour supporter un contenu URL
- l'ajout des balises <label> aux éléments <source>, <value>
- la modification des balises de type LANGSTRING pour avoir des langues sous forme d'URLs
- la nouvelle structure du schéma XSD avec l'inclusion des valeurs de vocabulaire sous forme de fichiers distincts dans le dossier scolomfr/vocab
